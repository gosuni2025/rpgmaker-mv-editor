# 3D モード

## 概要

3D モードは RPG Maker MV の 2D マップを **HD-2D スタイルのパースペクティブビュー**でレンダリングする機能です。
Three.js WebGL レンダラーを使用し、既存の RPG Maker MV のタイル/キャラクタースプライトを 3D 空間でレンダリングします。

![Big City 3D モード](../images/3d-mode/big-city-3d.png)

---

## 有効化の方法

### エディターで 3D ビューを見る

ツールバーの **3D ボタン**をクリックすると、エディターキャンバスが 3D ビューに切り替わります。

### ゲームで 3D モードを実行する

マップインスペクターで **3D モード**チェックボックスを有効にすると、そのマップでゲームプレイ時に自動的に 3D モードで開始されます。

> ゲーム内でも 2D ↔ 3D の切り替えが可能です。イベントコマンドまたはプラグインコマンドを通じてランタイム中に切り替えられます。

## 3D カメラ操作

### エディターで

3D モードが有効になると、マップキャンバスの右上に **gizmo (軸インジケーター)** が表示されます。

| 操作 | 動作 |
|------|------|
| マウスドラッグ | カメラ回転 (yaw) |
| マウスホイール | ズームイン/アウト |
| 右クリックドラッグ | カメラパン |

### ゲームで

[TouchCameraControl プラグイン](../plugins/touch-camera.md)が有効な場合:

| 操作 | 動作 |
|------|------|
| タッチ/ドラッグ | カメラ yaw 回転 |
| 2 本指ピンチ | ズームイン/アウト |
| マウスホイール | ズームイン/アウト |

## 3D レンダリングシステム

### HD-2D ビルボードスプライト

3D モードでは、キャラクターとイベントが**ビルボードスプライト**方式でレンダリングされます。カメラが回転してもスプライトは常にカメラに向き、カメラの向きに応じてキャラクタースプライトの方向 (フレーム) が自動的に補正されます。

### タイルレンダリング

- タイルマップは Three.js の `BufferGeometry` ベースで GPU 上で直接レンダリング
- 上層タイル (Upper layer, z=4) がキャラクター (z=3) を正常に隠す (屋根効果)
- 前後の順序は `renderOrder` で決定

### 3D タイルレイヤー (EXT)

マップインスペクターの **3D タイルレイヤー**オプションを有効にすると、上層タイルの Z 軸の高さが反映され、建物の屋根などが実際に高く見える効果を演出します。

## スカイボックス

![Lava Cave 3D ライティング](../images/3d-mode/lava-cave-lighting.png)

3D モードでは、マップ背景として**パノラマスカイボックス**を設定できます。

### 設定方法

マップインスペクター → **背景**セクション:

1. **Sky Sphere** ボタンをクリック
2. パノラマ画像ファイルを選択 (Equirectangular パノラマ PNG)
3. `img/skybox/` フォルダに画像を配置

対応画像形式: `img/skybox/ファイル名.png`

詳細は [SkyBox プラグインドキュメント](../plugins/skybox.md)を参照してください。

## ライティングシステム

3D モードでは、動的ライティングがリアルタイムでレンダリングされます。

- ポイントライト — 円形に広がる光
- アンビエントライト — 全体的な輝度調整
- 影の計算 — ShadowAndLight.js プラグインが処理

Lava Cave マップのように溶岩地形では、オレンジ/黄色のポイントライトが劇的な雰囲気を演出します。

詳細は[マップエディタ — ライティングシステム](02-map-editor.md#ライティングシステム-ext)を参照してください。

## 技術的な注意事項

### Three.js レンダリングの注意点

3D モードには以下の特性があります:

- **depthTest 無効**: すべてのオブジェクトが `depthTest: false` に設定されているため、前後の順序は position.z ではなく `renderOrder` のみで決定されます。
- **Y-flip 補正**: Mode3D のプロジェクション行列が Y 軸を反転 (`m[5] = -m[5]`) するため、オーバーレイオブジェクトには CanvasTexture の Y-flip 補正が必要です。
- **カメラ yaw ソート**: 3D モードでは `_sortChildren()` が `depth = x*sin(yaw) + y*cos(yaw)` の式でソートし、カメラが回転しても正しい前後順序を維持します。

### デュアルランタイム構造

プロジェクトフォルダには 2 つのランタイムが共存します:

| ファイル | ランタイム | 用途 |
|------|--------|------|
| `index.html` | PIXI.js | オリジナル RPG Maker MV 互換 |
| `index_3d.html` | Three.js | エディター 3D モード / プレイテスト |

オリジナルの RPG Maker MV Steam 版で開く/プレイテストする場合は `index.html` (PIXI) を使用します。
